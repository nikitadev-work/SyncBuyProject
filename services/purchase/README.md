# Documentation for **Purchase** microservice

Purchase service управляет совместными покупками: хранит покупки, участников и задачи (кто что добавил и на какую сумму), следит за жизненным циклом покупки.

### Состояния:

- ACTIVE — можно добавлять/редактировать задачи и участников

- LOCKED — изменения запрещены; на этой стадии уже можно запускать расчёт и оплаты

- FINISHED — всё оплачено, расчёты закрыты, покупка неизменна навсегда
